# UI 设计指南

> **设计类型**: App 设计（健康习惯打卡工具）
> **确认检查**: 本指南适用于可交互的个人健康管理应用。

> ℹ️ Section 1-2 为设计意图与决策上下文。Code agent 实现时以 Section 3 及之后的具体参数为准。

## 1. Design Archetype (设计原型)

### 1.1 内容理解

- **目标用户**: 关注个人健康的女性用户（Stacey），需要记录生理期、排泄、体重和运动等私密且日常的数据。使用场景多为晨起或睡前，心理状态期望轻松、无压力。
- **核心目的**: 降低记录门槛，通过可视化的月度视图建立成就感，方便回顾历史数据。
- **期望情绪**: 清新、愉悦、掌控感、私密安心。
- **需避免的感受**: 医疗化的冰冷感、复杂的操作负担、沉重压抑的视觉风格。

### 1.2 设计语言

- **Aesthetic Direction**: "Daily Joy"（日常悦己）。结合参考图的清晰结构，但将深色模式反转为明亮的白色基调，强调“椭圆标记”这一核心视觉特征，使其像贴纸一样可爱而非枯燥的数据点。
- **Visual Signature**: 
  1. **胶囊化日期标记**：日历中的日期数字被柔和的彩色椭圆包裹，多标记垂直堆叠如糖果。
  2. **纯净白底**：全局纯白背景，依靠色彩区分功能区块。
  3. **柔和圆角**：所有卡片和按钮采用大圆角（Pill Shape），消除锐利感。
- **Emotional Tone**: 轻盈（Lightweight）、亲和（Friendly）。
- **Design Style**: **Rounded 圆润几何** — 强调 pill 按钮和柔和阴影，配合高饱和度的马卡龙色系，营造无压力的记录体验。
- **Application Type**: Tool / Personal Dashboard - 单页应用，聚焦核心打卡流。

## 2. Design Principles (设计理念)

1. **One-Tap Joy**: 对于简单项（上厕所/大姨妈），点击即反馈，减少思考，用即时色彩反馈提供多巴胺奖励。
2. **Visual Stacking**: 将抽象的“多事件”转化为具象的“垂直堆叠椭圆”，利用色彩编码（绿/粉/蓝/橙）建立直观的记忆索引。
3. **Clean Canvas**: 保持背景的绝对干净（White），让用户的打卡数据成为页面上唯一的色彩主角，避免装饰性元素干扰数据阅读。
4. **Gentle Interaction**: 所有的交互动效（弹窗、删除、切换）应柔和流畅，避免生硬的跳变，符合健康管理类的舒缓节奏。

## 3. Color System (色彩系统)

> **配色设计理由**：基于“四种健康习惯”的功能属性，定义四组高辨识度但低攻击性的马卡龙色系。背景严格保持纯白，通过色块本身传递信息。

### 3.1 主题颜色

| 角色 | CSS 变量 | Tailwind Class | HSL 值 | 设计说明 |
| :--- | :--- | :--- | :--- | :--- |
| bg | `--background` | `bg-background` | `hsl(0 0% 100%)` | **纯白背景**，营造干净、私密的画布感 |
| card | `--card` | `bg-card` | `hsl(0 0% 100%)` | 卡片背景同背景色，通过阴影或边框区分层级 |
| text | `--foreground` | `text-foreground` | `hsl(222 47% 11%)` | 深灰黑，保证在白色背景上的高可读性 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | `hsl(220 9% 46%)` | 次要文字，用于未选中日期或非关键信息 |
| primary | `--primary` | `bg-primary` | `hsl(222 47% 11%)` | **主操作色**（如月份切换按钮），使用深色以压住阵脚 |
| primary-fg | `--primary-foreground` | `text-primary-foreground` | `hsl(0 0% 100%)` | 主操作文字色 |
| accent | `--accent` | `bg-accent` | `hsl(220 14% 96%)` | 交互反馈色（Hover 态），极淡的灰蓝色 |
| accent-fg | `--accent-foreground` | `text-accent-foreground` | `hsl(222 47% 11%)` | Accent 区域文字色 |
| border | `--border` | `border-border` | `hsl(220 13% 91%)` | 极淡的边框，用于分割日历格子或列表项 |

### 3.2 语义颜色（核心打卡标识）

> **强制规则**：这四个颜色专用于日历内的“椭圆标记”及对应的功能按钮背景，必须保持高饱和度但明度适中（马卡龙质感），确保在白色背景下清晰可见且不刺眼。

| 用途 | CSS 变量 (示例) | HSL 值 | 对应功能 | 视觉描述 |
| :--- | :--- | :--- | :--- | :--- |
| Success/Gut | `--color-gut` | `hsl(142 76% 36%)` | 上厕所 | **清新绿**：代表通畅、轻松 |
| Period | `--color-period` | `hsl(340 82% 52%)` | 大姨妈 | **柔粉色**：代表女性健康、温和 |
| Weight | `--color-weight` | `hsl(217 91% 60%)` | 体重 | **天空蓝**：代表数据、理性、冷静 |
| Sport | `--color-sport` | `hsl(28 85% 54%)` | 运动 | **活力橙**：代表能量、燃烧、动态 |

*注：代码实现时，这些语义色将直接应用于 `bg-[hsl(...)]` 或自定义 class，不纳入 shadcn 标准 token 以防止冲突，但需保持色调一致。*

### 3.3 Topbar/Header 设计策略

**背景策略**：
- 使用 `bg-background` (纯白)，不设独立背景色，与页面融为一体。
- 底部添加 `border-b border-border` 细线，仅做功能性分隔，不破坏整体通透感。

**文字与图标**：
- 月份标题：`text-foreground` + `font-bold`，字号放大，作为视觉重心。
- 切换箭头：`text-muted-foreground`，Hover 时变为 `text-foreground` 并伴随轻微位移。

## 4. Typography (字体排版)

- **Heading**: `Inter`, `system-ui`, sans-serif
  - *策略*: 现代无衬线，数字显示清晰。月份标题使用 `text-2xl` 或 `text-3xl`，加粗。
- **Body**: `Inter`, `system-ui`, sans-serif
  - *策略*: 默认字重 `font-normal`，行高 `leading-relaxed` 保证舒适阅读。
- **Calendar Numbers**: `font-medium`
  - *特殊处理*: 日历格子内的数字若被椭圆背景包裹，文字颜色自动适配（通常白色或深色，取决于背景明度，本方案中马卡龙色背景配白色文字更显精致，或深色文字保可读性，建议：绿色/蓝色背景配白字，粉色/橙色背景配深字或白字需测试对比度。**修正策略**：为确保统一性和“贴纸感”，所有椭圆背景内的文字统一使用 `text-white`，并微调背景色明度确保对比度 > 3:1）。
- **字体导入**: 
  ```css
  @import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap');
  ```

## 5. Layout Strategy (布局策略)

### 5.1 结构方向

**导航策略**：**None (内嵌式导航)**
- 理由：单页应用，月份切换作为 Header 的一部分直接集成，无需独立侧边栏或顶部导航栏，最大化内容展示区域。

**页面架构特征**：
- **垂直流式布局**：Header (月份) → Calendar Grid (核心视觉区) → Quick Actions (快捷入口) → Daily Log (详细记录)。
- **容器限制**：移动端全屏，桌面端限制 `max-w-md` 或 `max-w-lg` 居中，模拟手机应用体验，避免在大屏上过度拉伸导致视觉松散。

### 5.2 响应式原则

**断点策略**：
- **Mobile (<640px)**: 日历格子自适应填充，按钮单列或 2x2 紧凑排列。
- **Desktop**: 保持手机比例居中，左右留白，防止操作热区过远。

**内容密度**：
- 日历区：7 列固定，行高根据月份动态调整，确保每个格子内有足够空间容纳 4 个垂直堆叠的椭圆。
- 记录区：列表项单列展示，删除按钮置于右侧，易于触摸。

## 6. Visual Language (视觉语言)

**形态特征**：
- **椭圆标记 (The Pill)**：这是本设计的灵魂。日历格子里的标记不是小圆点，而是带有圆角的短条（`rounded-full` 或 `rounded-md`），高度约 6-8px，宽度随格子宽度自适应或固定比例。
- **大圆角容器**：功能按钮、记录卡片均采用 `rounded-2xl` 或 `rounded-3xl`，消除锐角，传递亲和力。
- **无边框主义**：除必要的分割线外，尽量通过背景色差（极淡）和阴影来区分层级，减少线条带来的割裂感。

**装饰策略**：
- **极简主义**：除了必要的功能图标（厕所、水滴、秤、跑鞋），不添加任何多余的装饰图案。
- **色彩即装饰**：利用四个功能色的自然分布，让日历本身成为一幅随时间变化的彩色图表。

**动效原则**：
- **微弹跳 (Bounce)**：点击打卡按钮时，按钮有轻微的缩放反馈；新生成的椭圆标记在日历中出现时，有一个小小的 `scale-in` 动画，增加趣味性。
- **平滑过渡**：月份切换时，日历网格整体向左/右滑动淡出，新月份滑入，避免生硬刷新。

**可及性保障**：
- 椭圆标记内的文字颜色必须与背景色保持足够对比度（若背景较浅，改用深色文字或增加背景不透明度）。
- 删除操作需有二次确认或明显的 Toast 提示，防止误删。

## 7. Component Principles (组件原则)

**状态完整性**：
- **Calendar Cell**: 
  - Default: 白底，灰字。
  - Selected: 浅灰背景 (`bg-accent`) 或 细边框高亮。
  - Has Data: 内部垂直堆叠彩色椭圆。
- **Action Button**: 
  - Default: 对应功能色的浅色背景 + 深色文字/图标，或纯色背景 + 白字。
  - Hover: 亮度微调。
  - Active: 缩放 0.95。
- **Dialog (Weight/Sport)**: 
  - 遮罩层 `bg-black/40`。
  - 弹窗主体 `bg-white rounded-2xl shadow-xl`。
  - 输入框/选择器需有大尺寸点击区域。

**层级清晰**：
- 视觉权重：彩色椭圆标记 > 功能按钮 > 日期数字 > 辅助文字。
- 确保“当日记录”区域的删除按钮虽然小，但颜色明确（如红色或灰色 Hover 变红），位置固定右侧。

**一致性**：
- 所有涉及“颜色”的地方，严格复用 Section 3.2 定义的 HSL 值，严禁随意取色。
- 所有圆角保持统一体系（如按钮 `rounded-full`，卡片 `rounded-2xl`）。

## 8. Design Signature (设计签名)

**核心识别特征**：
1. **“糖果堆叠”日历**：日期格子里垂直排列的彩色椭圆条，像糖果一样直观展示当天的健康完成度，顺序严格固定（绿→粉→蓝→橙）。
2. **纯净白底舞台**：没有任何杂色背景干扰，让用户的每一次打卡都像是在洁白的画布上留下一抹色彩。
3. **直觉化色彩编码**：绿=通畅，粉=生理，蓝=数据，橙=活力，无需文字即可通过颜色快速扫视月度状态。

**应避免**：
- ❌ 避免使用深色模式（除非用户强制切换，否则默认保持明亮的白色主题）。
- ❌ 避免在日历格子内使用复杂的图表或折线，保持“有/无”的二元直观性。
- ❌ 避免按钮样式千篇一律，四个功能按钮应通过颜色暗示其内容属性。
- ❌ 避免椭圆标记过小导致无法识别，需保证在移动端的最小可触控/可视尺寸。
